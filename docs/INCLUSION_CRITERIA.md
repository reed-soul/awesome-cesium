# Awesome Cesium - Inclusion Criteria

This document defines the formal criteria for including resources in the Awesome Cesium list. All submissions should meet these standards.

## Core Principles

1. **Quality over Quantity** - Only include well-maintained, useful resources
2. **Direct Relevance** - Must extend, enhance, or integrate with CesiumJS
3. **Accuracy** - All links must work; descriptions must be accurate

---

## Inclusion Requirements

### For All Resources

| Criterion | Requirement |
|-----------|-------------|
| **Cesium Relationship** | Directly related to CesiumJS, 3D Tiles, or the Cesium ecosystem |
| **Documentation** | Clear README, docs, or usage examples |
| **Uniqueness** | Does not duplicate existing listed resources |

### For GitHub Repositories

| Criterion | Requirement | Exception |
|-----------|-------------|-----------|
| **Last Commit** | Within 24 months | Official/historical significance |
| **README** | Comprehensive README | - |
| **License** | Clear license information | - |
| **Functionality** | Works as described | - |

### For Documentation/Tutorials

| Criterion | Requirement |
|-----------|-------------|
| **Accuracy** | Content is current and correct |
| **Clarity** | Well-structured, readable |
| **Examples** | Includes practical examples |

### For Tools/Applications

| Criterion | Requirement |
|-----------|-------------|
| **Stability** | Functional and reliable |
| **Installation** | Clear setup instructions |
| **Value** | Solves real Cesium developer needs |

---

## Exclusion Criteria

Resources will **not** be included if they:

- Are abandoned (no commits > 24 months, unless officially significant)
- Have broken or inaccessible links
- Duplicate functionality of better-maintained alternatives
- Have no clear Cesium relationship
- Lack documentation or usage examples
- Are purely commercial with no free/open component

## CesiumJS Version Compatibility

- **Current CesiumJS**: 1.x (check [releases](https://github.com/CesiumGS/cesium/releases))
- When adding resources, note CesiumJS compatibility if known
- Resources marked Outdated may target older Cesium versions

---

## Maintenance

- **Link Check**: Weekly (automated)
- **Quality Audit**: Monthly (automated - checks repo activity)
- **Manual Review**: Quarterly for category structure
- **Criteria Updates**: As Cesium ecosystem evolves
